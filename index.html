// 01 — PENGANTAR

Mengapa Sistem Kontrol Otomatis di Kapal?

Kapal modern mengoperasikan ratusan parameter secara simultan. Tanpa sistem kontrol otomatis, engineer harus terus-menerus memonitor dan menyesuaikan setiap parameter — tidak mungkin dilakukan manusia sendiri selama 24 jam berlayar.

⚙️
Efisiensi
Operasi Optimal

Kontrol otomatis menjaga parameter pada titik efisiensi terbaik — bahan bakar hemat, output maksimal, usia mesin lebih panjang. Fuel injection butuh presisi ±0.1 ms.

🛡️
SOLAS Compliance
Keselamatan Kapal

SOLAS Chapter II-1 mewajibkan alarm & shutdown otomatis. Respons manusia (3–10 detik) terlalu lambat mencegah kerusakan fatal dibanding PLC (10–100 ms).

🌍
MARPOL / IMO Tier III
Regulasi Emisi

MARPOL Annex VI memerlukan kontrol real-time scrubber (pH), EGR valve, dan SCR untuk memenuhi batas NOx/SOx. Hanya otomasi yang mampu melakukan ini.

📡
IAS / SCADA
Integrasi Terpusat

IAS mengumpulkan data 200+ parameter ke satu layar di ECR dan bridge. Engineer memantau seluruh kapal tanpa harus turun ke engine room.

● CLOSED-LOOP (Feedback)
SP → CONTROLLER → PLANT → OUTPUT
↑_________ FEEDBACK _________↑

Error = SP − PV. Controller terus menyesuaikan output agar error = 0. Lebih akurat, lebih aman. Digunakan di mayoritas sistem kapal.

○ OPEN-LOOP (Tanpa Feedback)

(Tidak ada feedback)

Tidak ada koreksi otomatis jika output meleset. Digunakan untuk urutan sederhana (sequential step) dimana hasil langkah satu sudah pasti sebelum lanjut ke langkah berikutnya.

// 02 — SEQUENTIAL CONTROL

Apa Itu Sequential Control System?

Sequential control (kontrol sekuensial) adalah sistem di mana operasi dilaksanakan dalam urutan langkah-langkah yang sudah ditentukan — setiap langkah baru bisa dimulai setelah langkah sebelumnya selesai atau kondisi tertentu terpenuhi.

// Definisi Sequential Control

Sequential Control adalah metode kontrol yang menjalankan serangkaian operasi atau tindakan secara berurutan (step-by-step), di mana:

  • Kondisi (Condition) — setiap langkah baru dimulai bila kondisi tertentu terpenuhi
  • Interlock — pengaman agar langkah tidak bisa dilewati jika kondisi belum siap
  • Timer — langkah dapat dipicu oleh waktu (misal: purge 30 detik)
  • State Machine — sistem selalu berada dalam satu "state" tertentu
// Karakteristik Utama
🔢
STEP-BASED
Berbasis langkah diskrit, bukan nilai kontinu
🔒
INTERLOCK
Keamanan berlapis antar langkah
⏱️
TIMER
Langkah dikontrol waktu atau event
🔄
STATE MACHINE
Satu state aktif di satu waktu

// 03 — PERBANDINGAN

Sequential vs Continuous Control

Memahami perbedaan dua metode kontrol utama ini adalah fondasi untuk mengerti semua sistem otomasi di kapal. Keduanya bekerja bersama-sama, bukan saling menggantikan.

Aspek Sequential Control Continuous Control (PID)
Prinsip Kerja STEP Operasi berurutan, satu langkah menunggu langkah sebelumnya selesai MATH Hitung error secara terus-menerus, koreksi proporsional terhadap deviasi
Jenis Sinyal Diskrit — Digital (ON/OFF, TRUE/FALSE) Kontinu — Analog (4-20mA, 0-10V, nilai desimal)
Logika IF-THEN, State Machine, Interlock, Timer Formula matematika: PID = P + I + D
Contoh Kapal Start engine, boiler startup, purifier auto-start, ballast sequence Suhu air pendingin, tekanan uap, RPM mesin, pH scrubber
Output PLC Digital Output (DO) — kontaktor, solenoid, relay Analog Output (AO) — valve positioner, VFD speed ref
Bahasa PLC Ladder Diagram (LD), Sequential Function Chart (SFC) Function Block Diagram (FBD), Structured Text (ST)
Keunggulan Aman (interlock), deterministik, mudah di-troubleshoot Akurasi tinggi, respons cepat terhadap gangguan kontinu
Kekurangan Tidak bisa mengontrol nilai kontinu secara halus Tidak bisa mengelola urutan operasi yang kompleks
Kompleksitas ⭐⭐⭐ (logika state kompleks) ⭐⭐⭐ (tuning parameter butuh keahlian)
Digunakan Bersama? ✓ YA — Hampir semua sistem kapal gabungan keduanya!
// Sequential Only
  • Turning gear interlock
  • Fire alarm sequence
  • Bilge pump auto-start
  • CO₂ flooding interlock
  • Anchor windlass interlock
// Continuous (PID) Only
  • Autopilot heading (PID)
  • Fuel oil temperature
  • Scrubber pH control
  • Refrigeration superheat
  • Turbocharger boost pressure
// Hybrid (Sequential + PID)
  • Boiler: start (seq) + pressure (PID)
  • Fresh water gen: startup (seq) + salinity (PID)
  • Ballast: valve sequence + level PID
  • Purifier: startup sequence + speed PID
  • Main engine: start seq + RPM PID
// 04 — PLC & SEQUENTIAL LOGIC

Banyak yang salah mengira "PLC = Sequential". Ini tidak tepat. PLC adalah alat (hardware + software), sedangkan Sequential adalah salah satu metode kontrol yang bisa dijalankan di PLC.

// BAHASA PEMROGRAMAN PLC (IEC 61131-3)
LD
Ladder Diagram

Paling umum di kapal. Mirip diagram rangkaian listrik — mudah dibaca oleh engineer yang terbiasa dengan wiring diagram. Digunakan untuk sequential dan on/off logic.

Contoh: Pompa start sequence, interlock relay logic, alarm circuit
SFC
Sequential Function Chart

Dirancang khusus untuk sequential control. Menampilkan alur step-by-step dengan transisi kondisi antar langkah. Standar IEC 61131-3 untuk SFC.

Contoh: Boiler startup, main engine starting sequence, cargo pump sequence
FBD
Function Block Diagram

Ideal untuk loop kontrol PID dan operasi matematis. Setiap blok mewakili fungsi (PID, compare, arithmetic). Mudah dikonfigurasi secara visual.

Contoh: PID temperature control, cascade loop, ratio control
ST
Structured Text

Mirip bahasa Pascal/C. Paling fleksibel untuk algoritma kompleks (Fuzzy, MPC). Digunakan untuk kalkulasi matematis yang rumit.

Contoh: Fuzzy logic CPP blade, MPC power management, data logging

SFC menjamin bahwa purifier tidak bisa menerima oli sebelum bowl mencapai kecepatan penuh. Interlock ini mencegah oil carry-over yang bisa merusak sistem bahan bakar. Tidak mungkin dilakukan dengan PID murni.

// 05 — SIKLUS KERJA PLC

Cara PLC Bekerja — Scan Cycle

PLC bukan komputer biasa. Ia bekerja dalam siklus deterministik yang berulang terus-menerus dalam hitungan milidetik. Memahami siklus ini krusial untuk troubleshooting dan commissioning.

01 Input Scan

Baca semua sinyal dari sensor (suhu, tekanan, level, RPM) dan simpan di memori Input Image Table

~1–5 ms
02 Program Exec

Jalankan program (Ladder/SFC/FBD): hitung logika, jalankan PID, update state machine sequential

~5–50 ms
03 Output Scan

Update sinyal output ke aktuator: buka valve, nyalakan motor, kirim data ke IAS via fieldbus

~1–5 ms
04 Housekeeping

Diagnostik internal, komunikasi jaringan, reset watchdog timer. Jika fault → alarm ke IAS

~1–3 ms
↺ Siklus berulang 10–100 kali per detik (10–100 ms total scan time)
// Scan Time Kapal

Standar: 10–100 ms (10–100 siklus/detik)
Safety kritis (steering, fire pump): <10 ms
Slow process (ballast level): <500 ms cukup
Semakin pendek scan time = semakin cepat respons

// Watchdog Timer

Jika PLC tidak mereset watchdog dalam waktu tertentu (misal 200 ms), sistem dianggap hang dan PLC otomatis reset atau masuk safe state. Vital untuk keselamatan kapal.

// Input Image Table

PLC tidak membaca sensor langsung saat program berjalan — ia membaca snapshot (image) yang diambil di awal siklus. Ini menjamin konsistensi data dalam satu siklus scan.

// 07 — CONTOH SISTEM SEKUENSIAL

Implementasi Sequential Control di Kapal

Empat contoh nyata implementasi sequential control pada permesinan kapal — dari yang paling kritis (main engine) hingga yang paling sering ditemui (ballast).

A
Main Engine Start Sequence
Wärtsilä / MAN Energy Solutions — Sequential control WAJIB, tidak boleh manual
01 TURNING GEAR

Turning gear harus dilepas. Interlock: jika turning gear masih engage → mesin tidak bisa start

🔒 INTERLOCK: TG_DISENGAGED = TRUE
02 PRELUBE OIL

Pompa minyak lumas dijalankan terlebih dahulu untuk melumasi semua bearing sebelum ada gerakan

⏱️ TIMER: Pre-lube 60 detik
03 START AIR

Udara starting (25-30 bar) diinjeksikan untuk memutar mesin hingga RPM tertentu

📊 Cek: Oil pressure > 1.5 bar
04 FUEL ON

Bahan bakar diinjeksikan setelah RPM mencapai firing speed. Mesin mulai bertenaga sendiri

📊 Cek: RPM ≥ 40% nominal
05 MONITOR

IAS memantau semua parameter — suhu, tekanan, RPM, getaran selama warm-up

⏱️ Warm-up: 15-30 menit
B
Boiler Start-Up Sequence (BMS)
Burner Management System — Siemens / Honeywell
01 PRE-PURGE

Kipas (fan) memurnikan ruang bakar dari sisa gas sebelum ignition dimulai

⏱️ PURGE TIME: 30 detik (minimum)
02 IGNITION

Pilot burner menyala dengan transformer ignition. Percikan listrik menghasilkan api kecil

⚡ Ignition transformer: 10kV
03 FLAME CHECK

Detektor nyala (flame eye / UV scanner) memverifikasi api terbentuk dalam waktu 5 detik

⏱️ FLAME FAIL TIME: 5 detik max
04 MAIN FUEL

Setelah pilot flame terkonfirmasi, main fuel valve dibuka untuk api penuh

🔒 INTERLOCK: FLAME_OK = TRUE
05 PID PRESSURE

Kontrol tekanan uap beralih ke PID. Sequential selesai — continuous control mengambil alih

🔁 Handover ke PID boiler
// LOCKOUT: Flame Failure

Jika flame tidak terdeteksi dalam 5 detik (FLAME_FAIL), BMS secara otomatis menutup semua fuel valve dan mengunci sistem (LOCKOUT). Reset manual oleh engineer wajib sebelum startup ulang. Ini contoh interlock keselamatan SOLAS pada sequential control.

C
Ballast System Sequence
Ballast Water Management — Kongsberg / Wartsila
01
Valve Selection

Operator memilih tangki tujuan. IAS mengirim perintah ke PLC untuk membuka valve yang sesuai.

Interlock: Hanya valve yang dipilih yang terbuka. Valve lain terkunci tertutup.
02
Valve Confirm

PLC mengkonfirmasi valve sudah terbuka penuh melalui limit switch feedback sebelum memulai pompa.

Interlock: VALVE_OPEN_FEEDBACK = TRUE
03
Pump Start

Pompa ballast dijalankan setelah valve terkonfirmasi terbuka. DOL atau Star-Delta starter.

Cek: Motor running feedback, current normal
04
Level PID Control

Setelah pompa jalan, PID mengontrol level tangki. Sequential selesai, continuous control aktif.

Cascade: outer loop level, inner loop flow rate
05
Auto Stop

Pompa otomatis berhenti saat level mencapai setpoint. Valve menutup secara berurutan.

Shutdown: pump stop → valve tutup → reset

// 09 — AKTUATOR & ELECTRICAL STARTER

// 10 — STUDI KASUS

Analisa Kegagalan Sistem Sekuensial

Empat studi kasus nyata yang sering terjadi di kapal — analisis logika sequential, identifikasi penyebab, dan troubleshooting step-by-step.

⚠ Studi Kasus 1: Main Engine Gagal Start KRITIS
▼
// SYMPTOM

Mesin mendapat perintah start dari bridge, turning gear sudah dilepas, tapi mesin tidak cranking. IAS menampilkan "START FAIL — CHECK INTERLOCKS"

// SEQUENTIAL LOGIC ANALYSIS
Step 1: TURNING_GEAR_DISENGAGED = TRUE ✓
Step 2: LUBE_OIL_PRESS > 1.5 bar
  → CURRENT VALUE: 0.8 bar ✗ FAIL

RESULT: Sequence HALTED at Step 2
INTERLOCK: STARTING_AIR blocked
ALARM: "LOW LO PRESSURE — PRELUBE FAIL"
// ROOT CAUSE & TROUBLESHOOTING
01
Cek Pre-lube Pump

Pastikan pre-lube pump berputar. Dengarkan suara motor — jika diam, cek starter dan alarm motor.

02
Cek Pressure Gauge Manual

Bandingkan pembacaan IAS dengan gauge manual. Jika gauge manual normal → sensor tekanan rusak (false reading).

03
Cek LO Level & Filter

Level sump rendah atau strainer tersumbat dapat menyebabkan tekanan rendah. Periksa secara manual.

// Pelajaran Sequential

Interlock prelube pressure mencegah mesin start dengan pelumasan tidak memadai — mencegah kerusakan bearing dan crankshaft yang sangat mahal. Inilah nilai sequential control dalam keselamatan.

🔥 Studi Kasus 2: Boiler Flame Failure KRITIS
▼
// SYMPTOM

Alarm berbunyi: "BOILER FLAME FAILURE — LOCKOUT". Seluruh sistem boiler mati tiba-tiba. Steam pressure turun drastis. IAS menampilkan kondisi LOCKOUT.

// BMS SEQUENTIAL RESPONSE
Event: FLAME_DETECTOR = OFF
during STEP 3 (flame check window)

BMS Response:
→ CLOSE PILOT FUEL VALVE (immediate)
→ CLOSE MAIN FUEL VALVE (immediate)
→ CLOSE ATOMIZING STEAM VALVE
→ SET LOCKOUT FLAG = TRUE
→ Send ALARM to IAS
→ Require MANUAL RESET before restart
// PENYEBAB UMUM & TINDAKAN
Flame eye kotor/rusak → Bersihkan optik UV scanner, ganti bila perlu
Fuel oil terlalu dingin → Cek viscosity, pastikan preheater berfungsi
Atomizing steam rendah → Cek steam pressure untuk atomisasi HFO
Air/gas dalam bahan bakar → Purging line, cek fuel pump
Burner nozzle tersumbat → Bersihkan atau ganti nozzle
// PERHATIAN KESELAMATAN

Jangan reset lockout tanpa investigasi penyebab! SOLAS melarang "blind restart" boiler setelah lockout. Lakukan purge manual min. 5 menit sebelum coba nyala ulang.

💧 Studi Kasus 3: Ballast Tank Overflow PERINGATAN
▼

Selama pengisian ballast, pompa tidak berhenti meskipun level tangki sudah mencapai 98%. Air hampir meluap dari overflow pipe.

// ANALISA LOGIKA
Expected: IF LEVEL > 95% → PUMP_STOP
Actual: PUMP still running at 98%

Kemungkinan masalah:
1. Level sensor (radar) error/kotor
2. PLC tidak menerima sinyal level
3. Program logic salah (threshold wrong)
4. Valve interlock tidak aktif
// TINDAKAN DARURAT & PENCEGAHAN
!!
Matikan Pompa Manual

Segera hentikan pompa dari panel lokal. Tutup valve secara manual untuk mencegah overflow.

01
Verifikasi Level Sensor

Bandingkan dengan sounding manual. Jika sounding manual = 98% tapi IAS = 60% → sensor bermasalah.

02
Cek PLC Program

Review ladder diagram untuk interlock auto-stop. Cek apakah threshold ditulis dengan benar di program.

🔋 Studi Kasus 4: Generator Gagal Sinkronisasi PERINGATAN
▼

Generator bantu tidak bisa paralel ke bus bar. PMS menampilkan "SYNCHRONIZATION FAIL" setelah 3 percobaan. Hanya 1 genset yang beroperasi dengan beban penuh.

PMS Sequential — Synchronization Sequence:
Step 1: Start generator → RPM nominal
Step 2: Adjust governor → f = 50 Hz ✓
Step 3: Adjust AVR → V = 450V ✓
Step 4: Phase sync window ← FAIL HERE
  → dφ > 5° saat close attempt

Result: Circuit breaker tidak ditutup
Action: Retry sequence (max 3x)
After 3 fail: ALARM + Manual required
// PENJELASAN & SOLUSI

Sinkronisasi memerlukan frekuensi, tegangan, dan fase yang hampir identik antara generator baru dan bus bar yang sudah beroperasi.

▸ Cek governor response — AVR dan governor harus responsif cukup cepat untuk sinkronisasi window
▸ Cek VR (Voltage Regulator) — tegangan fluktuatif bisa mencegah sinkronisasi
▸ Manual sinkronisasi darurat — gunakan synchroscope manual jika PMS gagal
// Sequential Safety

PMS membatasi percobaan sinkronisasi (max 3x) untuk mencegah kerusakan akibat paralel "kasar" yang bisa menghasilkan lonjakan arus besar dan merusak winding generator.

// Sequential vs PID — Ingat Ini!

Sequential mengontrol urutan operasi (kapan sesuatu dimulai dan berhenti).
PID mengontrol nilai variabel (seberapa besar valve dibuka, seberapa cepat motor berputar).

Contoh boiler: Sequential menentukan kapan burner menyala. PID menentukan seberapa banyak bahan bakar yang dikirim untuk menjaga tekanan 7 bar.

// Electrical Starter — Ingat Ini!

Starter HANYA untuk motor listrik. Valve pneumatik, solenoid valve, servo valve, heater element dikontrol langsung oleh output PLC (4-20mA atau 24VDC) tanpa starter.

// Interlock — Ingat Ini!

Interlock adalah logika yang mencegah operasi berbahaya. Tanpa interlock prelube pressure, main engine bisa start tanpa pelumasan → bearing hancur. Interlock adalah nyawa sequential control.

🎯 SETPOINT

Definisi

Setpoint (SP) adalah nilai target yang ingin dicapai atau dijaga oleh sistem kontrol. Nilai ini ditetapkan oleh operator atau engineer berdasarkan kebutuhan operasional dan spesifikasi teknis mesin.

Contoh Setpoint di Kapal

  • Suhu air pendingin ME: SP = 85°C
  • Tekanan uap boiler: SP = 7 bar
  • Level tangki ballast: SP = 80%
  • pH scrubber washwater: SP = 6.5
  • RPM generator: SP = 1500 rpm (50 Hz)

Siapa yang Mengatur Setpoint?

Setpoint bisa diubah dari IAS console (ECR/Bridge) tanpa harus ke lapangan. Perubahan setpoint dilog secara otomatis untuk audit trail dan pelaporan.

📡 SENSOR

Fungsi Sensor dalam Kontrol

Sensor mengukur kondisi aktual proses dan mengubahnya menjadi sinyal listrik yang bisa dibaca PLC. Akurasi sensor sangat kritis — sensor yang salah menghasilkan kontrol yang salah.

Jenis Sensor di Kapal

  • Thermocouple / RTD PT100: Suhu (−200 hingga +1000°C)
  • Pressure Transmitter: Tekanan (4-20mA output)
  • Coriolis/Turbine Flow Meter: Laju aliran massa/volume
  • Radar Level Gauge: Level tangki ballast/cargo
  • pH Electrode: Keasaman scrubber / cooling water
  • Tachometer / Encoder: RPM dan posisi

Signal Standard

Standar industri: 4-20mA (paling umum, tahan noise), 0-10VDC, Profibus/HART untuk smart transmitter yang mengirim data diagnostik.

🖥️ CONTROLLER (PLC)

Apa itu PLC?

Programmable Logic Controller (PLC) adalah komputer industri yang dirancang untuk kontrol proses secara real-time. Berbeda dari komputer biasa, PLC dirancang tahan terhadap getaran, suhu ekstrem, dan interferensi elektromagnetik — kondisi yang umum di engine room kapal.

Merek PLC Umum di Kapal

  • Siemens SIMATIC S7-300/400/1500 — paling umum di kapal Eropa
  • ABB AC500 — banyak di sistem ballast dan power management
  • Omron CX / NX — sistem kecil dan medium
  • Allen-Bradley ControlLogix — kapal Amerika
  • Schneider Modicon M340 — sistem auxiliary

Perbedaan PLC vs DCS

PLC cocok untuk sistem lokal (boiler, purifier) dengan I/O terbatas. DCS (Distributed Control System) untuk sistem besar seperti DP, PMS kapal besar dengan ribuan I/O tersebar di seluruh kapal.

🔧 AKTUATOR

Jenis Aktuator di Kapal

  • Motor Listrik — pompa, fan, kompressor, winch. Butuh starter (DOL/Star-Delta/VFD)
  • Valve Pneumatik — dikontrol 4-20mA ke positioner. Tidak butuh starter
  • Valve Hidrolik — steering gear, CPP. Servo valve dikontrol ±10VDC
  • Solenoid Valve — dikontrol 24VDC dari DO PLC. Tidak butuh starter
  • Heater Element — dikontrol 4-20mA ke power controller. Tidak butuh starter

Penting: Starter Hanya untuk Motor!

Banyak taruna salah memahami bahwa semua aktuator butuh starter. Electrical starter hanya untuk motor listrik. Aktuator non-motor dikontrol langsung dari output analog (AO) atau digital (DO) PLC.

🧭 BRIDGE / ECR LEVEL 3

Fungsi Level 3

Level tertinggi dalam hierarki otomasi kapal. Operator dan perwira dapat memonitor dan mengontrol seluruh sistem kapal dari satu posisi tanpa harus turun ke engine room.

Sistem yang Ada di Bridge/ECR

  • Autopilot System — Kongsberg, Raytheon
  • DP Operator Console — Kongsberg K-Pos, Rolls-Royce
  • Engine Telegraph — komunikasi perintah kecepatan
  • Alarm Management System — semua alarm PLC dari IAS
  • ECDIS — Electronic Chart Display

Protokol Komunikasi

NMEA 2000 untuk navigasi, Ethernet/Modbus TCP untuk IAS, GMDSS untuk distress communication.

📊 IAS / SCADA — LEVEL 2

Apa itu IAS?

Integrated Automation System (IAS) adalah sistem supervisory yang mengintegrasikan data dari semua PLC dan sistem kapal ke satu platform terpusat. Setara dengan SCADA di industri darat.

Fungsi IAS di Kapal

  • Monitoring semua sensor dan parameter secara real-time
  • Alarm management dan logging (wajib SOLAS)
  • Trending dan analisa data historis
  • Remote setpoint dari ECR
  • Performance monitoring (SFOCo, heat balance)

Merek IAS Umum

Kongsberg K-Chief, ABB Onboard DC, Rolls-Royce IAS, Wartsila NACOS

Fieldbus ke PLC

Profibus DP (paling umum), DeviceNet, EtherNet/IP, Modbus RTU/TCP

🖥️ PLC LEVEL 1 — EKSEKUTOR

Peran PLC dalam Hierarki

PLC di Level 1 adalah eksekutor langsung perintah kontrol. Ia menerima setpoint dari IAS (Level 2), mengeksekusi program sequential dan PID, serta mengirim data balik ke IAS untuk monitoring.

Typical PLC di Kapal

  • Boiler PLC — mengontrol BMS sequence + steam pressure PID
  • Purifier PLC — auto-start sequence + bowl speed control
  • FWG PLC — startup sequence + salinity PID
  • Scrubber PLC — pH dosing PID + blower control
  • Steering PLC — rudder angle feedback loop + hydraulic control

Koneksi ke IAS

Via Profibus DP atau Modbus RTU. Setiap PLC mengirim status, alarm, dan nilai process variable ke IAS setiap siklus scan. Latency biasanya <100ms.

🔥 BOILER — HYBRID CONTROL

Sequential Phase (Startup)

Urutan startup boiler menggunakan BMS (Burner Management System): Purge → Pilot Ignition → Flame Detection → Main Fuel. Setiap langkah memiliki interlock dan timer yang ketat.

PID Phase (Normal Operation)

Setelah nyala, PID mengambil alih:

  • Steam Pressure PID — mengontrol posisi burner modulating valve untuk menjaga tekanan 7 bar
  • Feedwater PI — mengontrol feedwater pump speed berdasarkan level drum
  • Combustion Air PID — O₂ trimming untuk efisiensi pembakaran

Shutdown Sequential

Shutdown juga menggunakan sequential: Close main fuel → Purge → Fan off. Tidak boleh langsung mematikan semua sekaligus.

⚙️ MAIN ENGINE — HYBRID CONTROL

Sequential: Starting Sequence

  • Turning gear dilepas (interlock)
  • Pre-lube pump ON → pressure check
  • Starting air valve buka
  • Engine cranking hingga firing speed
  • Fuel injection ON

PID: Running Operation

  • Governor (RPM PID) — mengontrol fuel rack position untuk menjaga RPM sesuai setpoint
  • Fuel Injection Timing PID — presisi 0.1ms untuk efisiensi dan emisi
  • Cooling Water Temperature PID — fresh water cooler bypass valve
  • Turbocharger Boost PID — waste gate actuator control
💧 FRESH WATER GENERATOR — HYBRID

Sequential: Startup

  • Cek cooling water suhu >70°C (dari jacket water ME)
  • Vacuum pump ON → buat tekanan rendah
  • Feed pump ON → masukkan sea water
  • Monitor vacuum pressure mencapai <0.1 bar
  • Open distillate valve

PID: Operasi Normal

  • Conductivity/Salinity PID — mengontrol brine pump speed. Jika salinitas terlalu tinggi, brine level dinaikkan untuk meningkatkan evaporasi
  • Vacuum PID — mengontrol cooling water flow valve
  • Production Rate Feedforward — antisipasi perubahan suhu jacket water
🌀 OIL PURIFIER — HYBRID CONTROL

Sequential: Auto-Start (SFC)

  • STEP 0: IDLE — semua OFF
  • STEP 1: Fill bowl dengan fresh water (30 detik)
  • STEP 2: Motor ON, accelerate ke 7200 RPM
  • STEP 3: Konfirmasi RPM, buka oil feed valve
  • STEP 4: Normal separation operation
  • STEP 5 (fault): Safe stop sequence

Fuzzy Logic: Separation Control

Alfa Laval menggunakan Fuzzy Logic untuk mengoptimalkan separation efficiency berdasarkan sinyal akustik (noise sensor) dan back pressure. Aturan IF-THEN disesuaikan dengan karakteristik oli yang berbeda.

Interlock kritis: Oil feed valve TIDAK bisa dibuka sebelum RPM mencapai 7200. Jika dibuka sebelumnya, oli akan terlempar keluar dan merusak bowl.
